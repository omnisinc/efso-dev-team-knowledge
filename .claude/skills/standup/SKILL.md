# SU 議事録スキル

Gemini の書き起こし・まとめメモから SU 議事録を作成し、ユーザーの手修正をルール・辞書にフィードバックする。

## Phase 1: 議事録作成

### 1. 入力の受け取り

- ユーザーから Gemini の **書き起こし** と **まとめメモ** の一方または両方を受け取る
- どちらが提供されたか明示する（「書き起こしのみ」「まとめメモのみ」「両方」）

### 2. ルール・辞書の読み込み

以下を Read する:
- `.claude/rules/standup-minutes.md` — 作成ルール
- `.claude/rules/misrecognition-dict.md` — 誤認識辞書・ドメイン用語集
- `.claude/skills/standup/standup-template.md` — テンプレート
- 直近の議事録 1〜2 件 — フォーマットの参考

### 3. 議事録の作成

`standup-minutes.md` のルールに従って議事録を作成する。

**判断が曖昧な項目は必ずユーザーに確認する:**
- 決定事項に含めるかどうか
- アクションアイテムに含めるかどうか
- チケット番号の種別（`EFS-XXX` か `#XXX` か）
- 発言者の特定が困難な場合

確認は `AskUserQuestion` で選択肢形式にする。

### 4. ファイル書き出しとステージング

1. `standup/YYYY/MM/YYYY-MM-DD.md` に Write する
2. Bash で `git add` してステージングする（Phase 3 の diff ベースライン）

```bash
git add standup/YYYY/MM/YYYY-MM-DD.md
```

### 5. ユーザーに手修正を依頼

以下のメッセージを出力して待機する:

> 議事録を作成しました。内容を確認し、必要に応じて手修正してください。
> 修正が終わったら教えてください。差分からルール・辞書の更新を検討します。
> 修正なしの場合もその旨教えてください。

---

## Phase 2: ユーザーの手修正（スキルの外）

ユーザーがエディタ等で直接ファイルを編集する。スキルはユーザーの応答を待つ。

---

## Phase 3: フィードバック反映

ユーザーから「修正した」等の応答があったら開始する。「修正なし」の場合はこの Phase をスキップして終了する。

### 1. diff の取得

```bash
git diff -- standup/YYYY/MM/YYYY-MM-DD.md
```

差分がなければ「差分はありません」と報告して終了。

### 2. 変更の分析と分類

diff の各変更を以下のカテゴリに分類する:

| カテゴリ | 例 | 反映先 |
|---|---|---|
| 誤認識修正 | 「OS Zero」→「Auth0」 | `misrecognition-dict.md` 技術用語セクション |
| ドメイン用語の表記統一 | 「コア」→「Core」 | `misrecognition-dict.md` ドメイン用語セクション |
| 決定事項の追加/削除 | 項目が削除された → 判定基準が甘かった | `standup-minutes.md` 決定事項の判定セクション |
| アクションアイテムの追加/削除 | 個人報告から昇格させた項目が削除された | `standup-minutes.md` アクションアイテムの判定セクション |
| 構成・フォーマット変更 | セクション順序、インデント等 | `standup-minutes.md` or テンプレート |
| 一過性の内容修正 | 事実誤認の訂正、文言の微調整 | 反映不要 |

### 3. 更新の提案

カテゴリごとに更新案をまとめてユーザーに提示する。

- **辞書更新候補**: 追加するエントリを具体的に示す
- **ルール更新候補**: 現行ルールの該当箇所と、追加・修正する文言を示す
- **反映不要と判断したもの**: 理由とともにリストし、ユーザーが異論あれば対応

提案は `AskUserQuestion` で項目ごとに承認/却下を選べるようにする。

### 4. 承認後の適用

承認された項目について:
- `misrecognition-dict.md` を Edit で更新
- `standup-minutes.md` を Edit で更新
- 必要に応じてテンプレートも更新

### 5. 完了

- 更新した内容のサマリーを出力
- ステージングを解除して通常の状態に戻す（ユーザーが自分のタイミングでコミットできるように）

```bash
git reset HEAD standup/YYYY/MM/YYYY-MM-DD.md
```
