# SU 議事録の作成ルール

## テンプレート

`standup/templates/standup-template.md` に従うこと。セクション順序を変えない。

## ファイル配置

- パス: `standup/YYYY/MM/YYYY-MM-DD.md`
- 日付はミーティング実施日

## メンバー表記

- **`<name>-san`**: ただの言及・発言者の記録（デフォルトはこちら）
- **`@handle`**: その人にやるべき TODO が発生 / 確認のお願いがある場合のみ

## チケット番号

- JIRA チケット: `EFS-XXX`
- GitHub PR / Issue: `#XXX`
- 書き起こし中で「チケット251」「251番」等の表現があれば文脈から `EFS-XXX` か `#XXX` か判断する

## 全体共有事項のグルーピング

書き起こしの中で同じ話題に触れている発言が複数ある場合、話者ごとに分けず **話題ごとにまとめる**。

- 話題が **2 つ以上** ある場合は `### 見出し` でグルーピング
- 話題が **1 つだけ** の場合は見出しを付けず箇条書きで直接記載する
- 関連する発言・補足・質疑応答は同じグループにまとめる
- 話題がなければ `特になし` とだけ記載する
- 各メンバーの進捗報告から読み取れる内容を全体共有事項に重複してまとめない（例: タスク状況の整理）

## 決定事項の判定

以下に該当するものを「決定事項」に記載する:
- 「〜でいく」「〜に決まった」「〜で OK」等、方針が確定した内容
- 明確な合意が得られたもの
曖昧な検討中の話題は決定事項に入れない。アクションアイテムとして確認・調整が残っているものは、まだ確定していないため決定事項に含めない。

## アクションアイテムの判定

以下に該当するものを記載する:
- 「〜する」「〜やる」「〜出す」等、誰かが具体的にやると言った作業
- 形式: `- [ ] @who: 何を (期限があれば期限)`

「今日やること」と重複する場合でも、会議で明示的にコミットされたものはアクションアイテムに入れる。逆に、個人報告内の「今日やること」を機械的にアクションアイテムに転記しない。

## インデント

- リストのネストは **4 スペース** でインデントする

## 各メンバー報告

- 見出しは `### handle` 形式
- 欠席者の報告は書かない
- 全体共有中心で個別報告がなかった回は `(個別進捗報告なし — 全体共有中心の回)` と記載
- 「ブロッカーや相談」が特にない場合はセクションごと省略する（「特になし」と書かない）
- 「これまでやったこと」「これからやること」等のラベルに `**` (強調) を付けない

## 入力ソース

ユーザーからは **Gemini の書き起こし** と **Gemini のまとめメモ** の両方が提供される。

- **書き起こし**: 一次ソース。議事録の根拠はこちらに基づく
- **まとめメモ**: Gemini が自動生成した要約・次のステップ等が含まれる。参考情報として活用するが、鵜呑みにしない

### 作成時の方針

- 両方を突き合わせて内容を確認する。まとめメモだけ、書き起こしだけに頼らない
- まとめメモの「推奨される次のステップ」「アクションアイテム」等は、書き起こしの発言内容と照合し、実際に会議で合意・コミットされたものだけを採用する
- まとめメモにあるが書き起こしで裏付けが取れない内容は採用しない
- 書き起こしにあるがまとめメモに反映されていない内容も見落とさない
- 判断に迷う場合はユーザーに確認する

## Gemini 書き起こしからの変換時の注意

書き起こしは音声認識の精度が低い。表面的な文字列ではなく、文脈から発言の意図を読み取って議事録に反映する。まとめメモの方が正しく把握しているケースもあるため、両方を突き合わせて判断する。

### 無視すべきノイズ

- フィラー（「えっと」「あー」「まあ」等）や相槌（「はい」「。」）は無視する
- 発言者でない人の独り言・作業中の声がマイクに拾われることがある（例: 「よしよしよし」「よいしょ」の連続）。会議への発言ではないので無視する

### 誤認識の読み替え

書き起こしでは技術用語・人名・数字が頻繁に誤変換される。文脈から正しい意味を推定する。

- **技術用語**: 英語の開発用語がカタカナや漢字に誤変換される（例: 「プルリカ」→ Pull Request、「ギットアブアクション図」→ GitHub Actions）
- 議事録の出力でも、プロジェクト固有のドメイン用語は用語集に従って表記する（用語集は今後整備予定）
- **人名**: メンバー表記の正規化表と照合する。漢字変換やカタカナ変換で別の名前に見えることがある（例: 「3つ止めさん」→ mitsudome-san）
- **チケット番号**: 数字が前後の発話と結合して別の番号に見えたり、桁が増減したりする。まとめメモや文脈と照合して判断する

### 発話の断片化

一人の発言の途中に別の人の相槌や独り言が挿入され、文が分断されることがある。前後をつなげて一つの発言として読み取る。

### その他

- Gemini の「まとめ」「詳細」「推奨される次のステップ」はそのまま使わない。テンプレートのセクション構造に再構成する
- 推測で内容を補わない。書き起こしにない情報は追加しない
- 文脈から判断できない内容（チケット番号の種別、発言者の特定、決定なのか検討中なのか等）はユーザーに確認してから記載する
